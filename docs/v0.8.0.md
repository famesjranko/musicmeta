# v0.8.0 — Production Readiness

## Context

External review identified four high-value gaps: no OkHttp adapter (adoption blocker for Android), no offline cache fallback (table stakes for mobile), no bulk enrichment API (common use case), JitPack-only distribution (blocks corporate adoption). These are the four highest value-to-effort items; Flow-based progressive API was assessed and deliberately cut.

## Implementation Order

1. **OkHttp adapter** — creates the new module needed before Maven Central config
2. **Stale cache** — independent, should exist before bulk (so bulk inherits offline support)
3. **Bulk enrichment** — depends on engine interface, simple sequential impl
4. **Maven Central publishing** — needs all modules to exist first

## Key Design Decisions

1. **No `CACHE_FIRST` mode** — needs background refresh architecture, out of scope. `STALE_IF_ERROR` covers the critical offline case.
2. **Stale only for `Error`/`RateLimited`** — not for `NotFound`. NotFound means "provider searched and found nothing", stale would be misleading.
3. **Bulk is sequential** — real pipelining deferred. Sequential with rate limiter is 90% of the value.
4. **OkHttp adapter has no retry logic** — delegates to OkHttp interceptors. Documented difference from DefaultHttpClient.
5. **`getIncludingExpired` has default `= null`** — custom cache impls don't break, they just don't support stale serving.
6. **Flow-based progressive API deferred** — identity resolution blocks all emission, mergeable types can't emit until all providers finish. Marginal benefit vs complexity. Callers can split `enrich()` calls today.

### HttpClient Interface — All 12 Methods

The interface is at `musicmeta-core/src/main/kotlin/com/landofoz/musicmeta/http/HttpClient.kt`. All methods are `suspend`. Two families:

**Nullable methods** (return `null` on any error — simpler but no error detail):
```
fetchJson(url: String): JSONObject?
fetchJsonArray(url: String): JSONArray?
fetchBody(url: String): String?
fetchRedirectUrl(url: String): String?           // follows redirects, returns final URL
postJson(url: String, body: String): JSONObject?
postJsonArray(url: String, body: String): JSONArray?
```

**HttpResult methods** (preserve exact error type — 429 vs 404 vs 500):
```
fetchJsonResult(url: String): HttpResult<JSONObject>
fetchJsonArrayResult(url: String): HttpResult<JSONArray>
postJsonResult(url: String, body: String): HttpResult<JSONObject>
postJsonArrayResult(url: String, body: String): HttpResult<JSONArray>
```

All return types use `org.json.JSONObject` / `org.json.JSONArray`. The OkHttp adapter must depend on `org.json` for parsing.

### HttpResult Sealed Class

At `musicmeta-core/src/main/kotlin/com/landofoz/musicmeta/http/HttpResult.kt`:

```kotlin
sealed class HttpResult<out T> {
    data class Ok<T>(val body: T, val statusCode: Int = 200) : HttpResult<T>()
    data class ClientError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
    data class ServerError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
    data class RateLimited(val retryAfterMs: Long? = null) : HttpResult<Nothing>()
    data class NetworkError(val message: String, val cause: Throwable? = null) : HttpResult<Nothing>()
}
```

Map OkHttp responses to these. The status code ranges in `DefaultHttpClient` are: 429 → `RateLimited`, 400-499 → `ClientError`, 500-599 → `ServerError`, 200-299 → `Ok`. Parse `Retry-After` header as seconds, multiply by 1000 for `retryAfterMs`.

### DefaultHttpClient Behavior to Match (or Deliberately Not Match)

At `musicmeta-core/src/main/kotlin/com/landofoz/musicmeta/http/DefaultHttpClient.kt` (289 lines):

**Must match:**
- Sets `User-Agent`, `Accept: application/json`, `Accept-Encoding: gzip` on every request
- `fetchRedirectUrl()` disables redirect following to capture the `Location` header from 3xx responses. Used by Cover Art Archive which returns 307 redirects to image CDN URLs.
- POST methods set `Content-Type: application/json` and write body as UTF-8
- Error body read from `errorStream` (not `inputStream`) for error responses — OkHttp handles this automatically via `response.body?.string()`
- All methods wrapped in `withContext(Dispatchers.IO)`

**Deliberately different (document in KDoc):**
- `DefaultHttpClient` has a `get()` method with 3 retries, exponential backoff (2s/4s/8s base + 25% jitter), and `Retry-After` header parsing with 120s max. The nullable methods (`fetchJson`, `fetchBody`, etc.) go through this retry path. The OkHttp adapter should NOT replicate this — OkHttp users add retry interceptors.
- `DefaultHttpClient` manually decompresses GZIP via `GZIPInputStream`. OkHttp handles transparent decompression automatically (don't set `Accept-Encoding: gzip` manually — OkHttp adds it and handles decompression).

### InMemoryEnrichmentCache — Key Detail

At `musicmeta-core/src/main/kotlin/com/landofoz/musicmeta/cache/InMemoryEnrichmentCache.kt` line 21:

```kotlin
if (clock() > entry.expiresAt) { entries.remove(key); return null }
```

This line eagerly deletes expired entries on read. For stale-while-revalidate, change to just `return null` **without** `entries.remove(key)`. The expired entry stays in the map and is available via `getIncludingExpired()`. Memory pressure is handled by LRU eviction (max 500 entries, managed by `LinkedHashMap` with accessOrder=true).

The cache uses `Mutex` for thread safety — `getIncludingExpired()` must also use `mutex.withLock`.

Cache key format: `"$entityKey:$type"` (line 49).

### Distinguishing Error from NotFound for Stale Serving

`ProviderChain.resolve()` preserves the failure type. When a provider returns `RateLimited` or `Error`, the chain propagates that. When a provider returns `NotFound` (searched and found nothing), the chain returns `NotFound`. The stale logic should only serve stale for `Error` and `RateLimited`, not `NotFound` — because NotFound means "the data doesn't exist" not "the network failed."

Check with `is EnrichmentResult.Error || is EnrichmentResult.RateLimited`. Timeout errors arrive as `EnrichmentResult.Error` with `errorKind = ErrorKind.TIMEOUT` (set at line 90), so they're already covered.
