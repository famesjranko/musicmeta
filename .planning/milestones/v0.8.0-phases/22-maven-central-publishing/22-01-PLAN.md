For JVM modules (core, okhttp):
```kotlin
import com.vanniktech.maven.publish.SonatypeHost

mavenPublishing {
    publishToMavenCentral(SonatypeHost.CENTRAL_PORTAL)
    signAllPublications()
    coordinates("com.landofoz", "<artifactId>", "0.8.0")
    pom { /* license, developer, scm */ }
}
```

For Android module:
```kotlin
import com.vanniktech.maven.publish.AndroidSingleVariantLibrary
import com.vanniktech.maven.publish.SonatypeHost

mavenPublishing {
    publishToMavenCentral(SonatypeHost.CENTRAL_PORTAL)
    signAllPublications()
    configure(AndroidSingleVariantLibrary(variant = "release", sourcesJar = true, publishJavadocJar = true))
    coordinates("com.landofoz", "musicmeta-android", "0.8.0")
    pom { /* license, developer, scm */ }
}
```

Key facts:
- Plugin replaces `maven-publish` — do NOT keep both
- Plugin auto-generates sources and javadoc jars for JVM targets
- `AndroidSingleVariantLibrary` replaces both `android { publishing { singleVariant() } }` and `afterEvaluate { publishing {} }`
- `java { withSourcesJar(); withJavadocJar() }` is NOT needed — plugin handles it
- Credentials go in `~/.gradle/gradle.properties` (never committed), with comment placeholders in project `gradle.properties`
