       ```kotlin
       package com.landofoz.musicmeta.http

       /**
        * Typed HTTP response that preserves status information for callers.
        * Unlike the nullable returns of [HttpClient.fetchJson], this captures
        * the specific failure mode so providers can react differently to
        * 404 (not found) vs 429 (rate limited) vs 500 (server error).
        */
       sealed class HttpResult<out T> {
           data class Ok<T>(val body: T, val statusCode: Int = 200) : HttpResult<T>()
           data class ClientError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
           data class ServerError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
           data class RateLimited(val retryAfterMs: Long? = null) : HttpResult<Nothing>()
           data class NetworkError(val message: String, val cause: Throwable? = null) : HttpResult<Nothing>()
       }
       ```

    2. Add to `HttpClient` interface:
       ```kotlin
       /**
        * GET request returning typed [HttpResult] with parsed JSON body.
        * Unlike [fetchJson] which returns null on any failure, this preserves
        * the specific HTTP status for precise error handling.
        */
       suspend fun fetchJsonResult(url: String): HttpResult<JSONObject>
       ```

    3. Implement `fetchJsonResult()` in `DefaultHttpClient`. This is a new method that does NOT use the existing `get()` method (which returns nullable String). Instead, write a dedicated implementation:
       - Open connection via `openConnection(url)`
       - On 200-299: read body, parse as JSONObject, return `HttpResult.Ok(json, responseCode)`
       - On 429: extract Retry-After header, return `HttpResult.RateLimited(retryAfterMs)`
       - On 401/403: return `HttpResult.ClientError(responseCode, body)`
       - On 400-499: return `HttpResult.ClientError(responseCode, body)`
       - On 500-599: return `HttpResult.ServerError(responseCode, body)`
       - On IOException: return `HttpResult.NetworkError(e.message ?: "Network error", e)`
       - On JSONException: return `HttpResult.NetworkError("JSON parse error: ${e.message}", e)` (treat parse failure as network-level since the body was malformed)
       - Run within `withContext(Dispatchers.IO)`
       - No retry logic in this method (keep it simple, providers handle retries)

    4. Add `fetchJsonResult()` to `FakeHttpClient`:

       ```kotlin
       private val httpResultResponses = mutableMapOf<String, HttpResult<JSONObject>>()

       fun givenHttpResult(urlContains: String, result: HttpResult<JSONObject>) {
           httpResultResponses[urlContains] = result
       }

       override suspend fun fetchJsonResult(url: String): HttpResult<JSONObject> {
           requestedUrls.add(url)
           val configured = httpResultResponses.entries.firstOrNull { url.contains(it.key) }
           if (configured != null) return configured.value
           // Fall back to existing fetchJson behavior for backward compatibility
           val json = fetchJson(url)
           return if (json != null) HttpResult.Ok(json) else HttpResult.NetworkError("No response configured")
       }
       ```
