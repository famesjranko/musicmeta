    2. Modify `resolveTypes()` to detect composite types and handle them. Before the existing fan-out logic, partition `types` into composite and standard:
    ```kotlin
    private suspend fun resolveTypes(
        request: EnrichmentRequest,
        types: Set<EnrichmentType>,
    ): Map<EnrichmentType, EnrichmentResult> = coroutineScope {
        val compositeTypes = types.filter { it in COMPOSITE_DEPENDENCIES }
        val standardTypes = types - compositeTypes.toSet()

        // Composite types need sub-type resolution first
        val compositeSubTypes = compositeTypes
            .flatMap { COMPOSITE_DEPENDENCIES[it] ?: emptySet() }
            .toSet() - standardTypes  // Don't double-resolve if caller also requested them

        // Resolve standard types + composite sub-types concurrently
        val allToResolve = standardTypes + compositeSubTypes
        val resolved = allToResolve.map { type ->
            async {
                val chain = registry.chainFor(type)
                val result = chain?.resolve(request) ?: EnrichmentResult.NotFound(type, "no_provider")
                type to filterByConfidence(result)
            }
        }.awaitAll().toMap().toMutableMap()

        // Synthesize composite types from resolved sub-types
        for (compositeType in compositeTypes) {
            val synthesized = synthesizeComposite(compositeType, request, resolved)
            resolved[compositeType] = synthesized
        }

        resolved
    }
    ```

    3. Add a private `synthesizeComposite()` method. For ARTIST_TIMELINE specifically:

    private fun synthesizeComposite(

The identity resolution stores its Metadata result in the outer `results` map under IDENTITY_TYPES (GENRE, LABEL, etc.). To get the metadata to the synthesizer, look for any Success result in the outer `results` that has Metadata data. BUT — resolveTypes doesn't have access to the outer results map.

    Better approach: Pass the identity metadata through the request's resolved identifiers. The enrichedRequest already carries merged identifiers. But the Metadata beginDate/endDate are on the Metadata data object, not on EnrichmentIdentifiers.

    Simplest approach: Make `resolveTypes` accept an optional `identityResult` parameter. In `enrich()`, after identity resolution, pass the identity result to `resolveTypes`. In `synthesizeTimeline`, extract the Metadata from the identity result.

    Actually, the cleanest approach per the CONTEXT.md "simple check" guidance: modify `resolveTypes` to also accept the identity resolution results map so composite synthesis can access metadata. Add parameter `identityResults: Map<EnrichmentType, EnrichmentResult> = emptyMap()`.

    Then in `synthesizeTimeline`:

    private fun synthesizeTimeline(
        request: EnrichmentRequest,
        resolved: Map<EnrichmentType, EnrichmentResult>,
        identityResults: Map<EnrichmentType, EnrichmentResult>,
    ): EnrichmentResult {
        // Find metadata from identity resolution (any IDENTITY_TYPES result with Metadata data)
        val metadata = identityResults.values.firstOrNull {
            it is EnrichmentResult.Success && it.data is EnrichmentData.Metadata
        }
        val discography = resolved[EnrichmentType.ARTIST_DISCOGRAPHY]
        val bandMembers = resolved[EnrichmentType.BAND_MEMBERS]

        val timeline = TimelineSynthesizer.synthesize(metadata, discography, bandMembers)
        return EnrichmentResult.Success(
            type = EnrichmentType.ARTIST_TIMELINE,
            data = timeline,
            provider = "timeline_synthesizer",
            confidence = ConfidenceCalculator.authoritative(),
        )
    }
    ```
