    GREEN phase — create TimelineSynthesizer.kt in engine/ package:

    ```kotlin
    package com.landofoz.musicmeta.engine

    import com.landofoz.musicmeta.*

    object TimelineSynthesizer {
        fun synthesize(
            metadata: EnrichmentResult?,
            discography: EnrichmentResult?,
            bandMembers: EnrichmentResult?,
        ): EnrichmentData.ArtistTimeline {
            val events = mutableListOf<TimelineEvent>()
            extractLifeSpanEvents(metadata, events)
            extractDiscographyEvents(discography, events)
            extractBandMemberEvents(bandMembers, events)
            val sorted = events.sortedBy { it.date }
            val deduped = sorted.distinctBy { "${it.date}:${it.type}" }
            return EnrichmentData.ArtistTimeline(deduped)
        }
    }
    ```

    Private helper methods:
    - `extractLifeSpanEvents`: If metadata is Success with Metadata data, read beginDate/endDate/artistType. For beginDate: if artistType is "Person" emit "born", else emit "formed". For endDate: if artistType is "Person" emit "died", else emit "disbanded". Default (null artistType) to Group behavior.
    - `extractDiscographyEvents`: If discography is Success with Discography data, iterate albums. Sort by year. Mark the first (earliest) album with non-null year as "first_album", rest as "album_release". Description = album title. relatedEntity = album title.
    - `extractBandMemberEvents`: If bandMembers is Success with BandMembers data, parse activePeriod strings. Format "YYYY-present" or "YYYY-YYYY" or "YYYY-?". For begin date, emit "member_joined" with description "{name} joined". For end date (if not "present" and not "?"), emit "member_left" with description "{name} left". relatedEntity = member name.

    Keep file under 100 lines. Pure functions, no side effects.
