sealed class HttpResult<out T> {
    data class Ok<T>(val body: T, val statusCode: Int = 200) : HttpResult<T>()
    data class ClientError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
    data class ServerError(val statusCode: Int, val body: String? = null) : HttpResult<Nothing>()
    data class RateLimited(val retryAfterMs: Long? = null) : HttpResult<Nothing>()
    data class NetworkError(val message: String, val cause: Throwable? = null) : HttpResult<Nothing>()
}

    Three private helpers (keeps each public method under 40 lines):

    1. `buildGetRequest(url: String): Request` — builds GET request with:
       - `User-Agent` header set to `userAgent`
       - `Accept: application/json` header
       - Do NOT set `Accept-Encoding` (OkHttp handles gzip automatically; setting it manually disables transparent decompression per HTTP-04)

    2. `buildPostRequest(url: String, body: String): Request` — builds POST request with:
       - Same headers as GET (`User-Agent`, `Accept: application/json`)
       - `Content-Type: application/json`
       - Body: `body.toRequestBody("application/json".toMediaType())`

    3. `toHttpResult(response: Response): HttpResult<String>` — maps status codes:
       - `response.code == 429` -> `HttpResult.RateLimited(retryAfterMs)` where `retryAfterMs = response.header("Retry-After")?.toLongOrNull()?.let { it * 1000 }`
       - `response.code in 400..499` -> `HttpResult.ClientError(response.code, response.body?.string())`
       - `response.code in 500..599` -> `HttpResult.ServerError(response.code, response.body?.string())`
       - `response.code in 200..299` -> `HttpResult.Ok(response.body?.string() ?: "", response.code)`
       - else -> `HttpResult.ClientError(response.code)`

    Implement all 10 methods. Every method uses `withContext(Dispatchers.IO) { ... }`. Every `execute()` call must use `response.use { }` to guarantee response body closure.

    **6 Nullable methods** (return null on any error):

    `fetchJson(url)`: Execute GET, if 2xx parse body as `JSONObject(text)`, catch `JSONException` and `IOException` -> return null.
    `fetchJsonArray(url)`: Same but `JSONArray(text)`.
    `fetchBody(url)`: Execute GET, if 2xx return body string, catch `IOException` -> return null.
    `fetchRedirectUrl(url)`: Execute GET using `noRedirectClient` (not the normal client). If 3xx, return `response.header("Location")`. If 2xx, return the original url. Otherwise null. Catch `IOException` -> return null.
    `postJson(url, body)`: Execute POST, if 2xx parse body as `JSONObject(text)`, catch `JSONException` and `IOException` -> return null.
    `postJsonArray(url, body)`: Same but `JSONArray(text)`.

    **4 HttpResult methods** (preserve error type):

    `fetchJsonResult(url)`: Execute GET, call `toHttpResult(response)`. If `Ok`, parse body string as `JSONObject` -> `HttpResult.Ok(jsonObj, code)`. If parse fails -> `HttpResult.NetworkError("JSON parse error: ...")`. Other results pass through. Catch `IOException` -> `HttpResult.NetworkError(msg, cause)`.
    `fetchJsonArrayResult(url)`: Same but `JSONArray`.
    `postJsonResult(url, body)`: Execute POST, same HttpResult mapping as fetchJsonResult.
    `postJsonArrayResult(url, body)`: Same but `JSONArray`.

    Add KDoc to the class documenting:
    - No built-in retry logic (unlike DefaultHttpClient). OkHttp users add retry via interceptors.
    - Gzip decompression handled transparently by OkHttp.
    - Timeouts inherited from the caller's OkHttpClient instance.
    - User-Agent set on every request.
